jQuery插件开发步骤

在实际开发工作中，总会碰到像滚动，分页，日历等展示效果的业务需求，扩展lQuery插件和方法作用是非常强大的，它可以节约大量的开发时间，在满足业务需求的同时，更有易于使用、维护、复用的优点。

这几天看了很多关于jQuery插件制作的文章，大同小异，在动手做了插件后，写一个摘记来备忘和记录。

一、jQuery插件基础

什么是jQuery插件？jQuery插件如何使用？

jQuery插件就是对jQuery原型对象扩展的一个方法，使用时调用jQuery对象的方法即可。

jQuery对象是如何获取这些方法的呢?

通过查看jQuery源码jQuery.fn = jQuery.prototype,也就是说jQuery.fn对象是jQuery内部定义的一个原型对象，jQuery的DOM操作方法都在jQuery.fn原型对象上定义的，然后jQuery对象就可以通过原型继承这些方法。

jQuery插件有哪几个种类？

1、类级别的插件开发

类级别的插件开发最直接的理解就是给jQuery类添加类方法，可以理解为添加静态方法。这里只简单的描述类级别插件的开发，主要对对象级别的插件开发做详细记录。

1.1 添加一个全局函数

jQuery.sayHello = function(){
alert(‘Hello，nice to meet you’); //调用时用jQuery.sayHello()或者$.sayHello()
}

1.2 添加多个全局函数

jQuery.sayHello = function(){
alert(‘Hello，nice to meet you’);
};
jQuery.sayBye = function(){
alert(‘Bye，Hope to meet you again’);
};

——————–我是分割线——————

jQuery.extend({
sayHello : function(){
alert(‘Hello，nice to meet you’);
},
sayBye : function(){
alert(‘Bye，Hope to meet you again’);
}
});

1.3 使用命名空间 （避免某些函数或变量名将于其他jQuery插件冲突）

jQuery.myPlugin = {
sayHello : function(){
alert(‘Hello，nice to meet you’);
},
sayBye : function(){
alert(‘Bye，Hope to meet you again’);
}
}; //采用命名空间的函数仍然是全局函数，调用时$.myPlugin.sayHello()

2、对象级别的插件开发

2.1 基础的jQuery插件

现在写一个很简单的jQuery插件，我们需要这个插件来改变标签内容的颜色，例如：

$.fn.changeStyle = function(colorStr){
this.css(“color”,colorStr);
}

$(“p”).changeStyle(“red”); //可以这样使用插件

当插件被调用时，插件内部的this代表的是当前使用该方法的jQuery对象，这样使用该方法的标签集合的样式都会改变，如例子内容会变成红色。

2.2 添加一个立即执行函数

为了保证该插件的变量和对象与其他插件的对象和变量不冲突，我们需要添加一个立即执行函数，它被包含在一对小括号里，让插件的代码在匿名函数的局部作用域运行，使外部无法访问函数内部的公共变量和对象命名空间。

(function($){
$.fn.changeStyle = function(colorStr){
this.css(“color”,colorStr);
}
}(jQuery));

这里把jQuery对象当形参传入，这样的话在这个插件内，jQuery对象就被映射为$，避免了$符号被其他库覆写，我们就可以寄无限制的用$符号来表示jQuery对象了。

2.3 满足链式调用的jQuery插件

很多时候，一个插件的意图仅仅是修改、筛选收集到的元素，并将他们继续传给下一个方法。一个jQuery插件应该保持jQuery的Chainability特性，要做到这一点，只需返回this关键字：

(function($){
$.fn.changeStyle = function(colorStr){
this.css(“color”,colorStr);
return this;
}
}(jQuery));

$(“p”).changeStyle(“red”).addClass(“some-class”); //可以这样使用插件

2.4 可以接受更多参数的jQuery插件

上面这种插件适用于接受比较少的参数，如果传入函数的参数比较多，我们可以定义一个参数对象：

(function($){
$.fn.changeStyle = function(options){
this.css(“color”,options.colorStr).css(“fontSize”,option.fontSize+”px”);
return this;
}

}(jQuery));

$(“p”).changeStyle({colorStr : “red” , fontSize : 20}); //可以这样使用插件

很多情况下，有一些配置参数是固定的，或者说是大多数情况下需要的，这样我们可以设置一些默认参数：

(function($){
$.fn.changeStyle = function(options){
var settings = $.extend(true, $.fn.changeStyle.defaults, options||{});
this.css(“color”,settings.colorStr).css(“fontSize”,settings.fontSize+”px”);
return this;
}

$.fnchangeStyle.defaults = {
   color : "blue";
   fontsize : 12 ;
   fontfamily : "微软雅黑";
   height : 50 ;
   width : 100 ;;
 };

}(jQuery));

$(“p”).changeStyle({colorStr : “red” , fontSize : 20}); //可以这样使用插件

本例中采用了深拷贝的extend()方法，这里不做介绍了。这个例子的颜色的字体大小不会是再是原来的默认值，而被修改成了红色和20
px的字体大小，而其他的选项则是默认值。

一个最基础的插件开发就是这样了。
